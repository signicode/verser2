# Draft Interface Moves

This draft tracks Host/Guest helpers that could move into `@signicode/verser-common`, be inlined, or be combined into more general common interfaces.

## Move to common

- [x] MOVE: source: `packages/verser2-host/src/lib/types.ts#VerserPeerRole` dest: `packages/verser-common/src/lib/registration.ts#VerserPeerRole` - moved the peer role union into common because it is part of the registration protocol sent by guests and parsed by hosts.
- [x] MOVE: source: `packages/verser2-host/src/lib/types.ts#VerserHostRegistrationRequest` dest: `packages/verser-common/src/lib/registration.ts#VerserRegistrationRequest` - moved and renamed the registration request type as a protocol-level contract shared by Host and Guest; Host keeps a backwards-compatible alias.
- [x] MOVE: source: `packages/verser2-host/src/lib/host-protocol.ts#parseRegistrationRequest` dest: `packages/verser-common/src/lib/registration.ts#parseRegistrationRequest` - moved registration JSON parsing and validation into common.
- [x] MOVE: source: `packages/verser2-host/src/lib/node-http2-verser-host.ts#RegistrationResponse` dest: `packages/verser-common/src/lib/registration.ts#VerserRegistrationResponse` - moved the registration response shape into common and exposed it as the shared Host-to-Guest response contract.
- [x] MOVE: source: `packages/verser2-guest-node/src/lib/types.ts#BrokerControlFrame` dest: `packages/verser-common/src/lib/registration.ts#VerserBrokerControlFrame` - moved the broker route-control frame type into common and normalized it as a discriminated protocol frame.
- [x] MOVE: source: `packages/verser2-host/src/lib/host-protocol.ts#decodeHeaderMap` dest: `packages/verser-common/src/lib/header-serialization.ts#decodeHeaderMap` - moved decoding of serialized `x-verser-headers` into common as part of the Broker/Host protocol contract.
- [x] MOVE: source: `packages/verser2-host/src/lib/host-protocol.ts#parseLeaseAcquireTimeoutMs` dest: `packages/verser-common/src/lib/protocol-headers.ts#parseLeaseAcquireTimeoutMs` - moved lease-acquire timeout parsing into common alongside the protocol header constant.
- [ ] MOVE: source: `packages/verser2-host/src/lib/utils.ts#ErrorResponse` dest: `packages/verser-common/src/lib/error-response.ts#VerserHttpErrorResponse` - move and rename the serialized HTTP error response shape into common.
- [ ] MOVE: source: `packages/verser2-host/src/lib/utils.ts#toErrorResponse` dest: `packages/verser-common/src/lib/error-response.ts#toVerserHttpErrorResponse` - move Host error body serialization into common so Guest-side parsing can share the same contract.
- [ ] MOVE: source: `packages/verser2-guest-node/src/lib/error-utils.ts#toVerserErrorCode` dest: `packages/verser-common/src/lib/error-response.ts#toVerserErrorCode` - move error code validation/coercion into common near `VerserErrorCode`.
- [ ] MOVE: source: `packages/verser2-guest-node/src/lib/error-utils.ts#errorFromBody` dest: `packages/verser-common/src/lib/error-response.ts#verserErrorFromResponseBody` - move the serialized error response parser into common and pair it with error response serialization.
- [ ] MOVE: source: `packages/verser2-guest-node/src/lib/error-utils.ts#getErrorMessage` dest: `packages/verser-common/src/lib/utils.ts#getErrorMessage` - use the existing common helper as a public shared utility instead of keeping local duplicates.
- [ ] MOVE: source: `packages/verser2-guest-js-common/src/lib/routes.ts#resolveRouteForHostname` dest: `packages/verser-common/src/lib/routing.ts#resolveRouteForHostname` - move exact hostname route selection into common if the exact-match policy is intended to be shared by all brokers/runtimes.
